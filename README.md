# 筆電 EC 電源狀態機模擬器

這是一個以 C 語言撰寫的命令列模擬器，用有限狀態機（Finite State Machine, FSM）模擬筆電 EC（Embedded Controller）在不同電源狀態之間的切換行為。

程式會用一個 8-bit 暫存器表示硬體供電狀態，並根據使用者輸入的事件切換系統狀態，例如開機、關機、闔上螢幕、打開螢幕與低電量。

## 功能特色

- 模擬筆電常見電源狀態：
  - `S5_OFF`：關機狀態
  - `S0_ON`：正常開機狀態
  - `S3_SLEEP`：睡眠狀態（Suspend to RAM）
- 透過 bit flag 模擬硬體電源：
  - Bit 0：CPU
  - Bit 1：RAM
  - Bit 2：螢幕
  - Bit 3：風扇
- 提供互動式事件輸入：
  - 電源鍵
  - 闔上螢幕
  - 打開螢幕
  - 低電量
- 每次狀態改變後會印出目前硬體供電狀態與暫存器值。

## 狀態轉移說明

| 目前狀態 | 事件 | 下一狀態 | 硬體供電變化 |
| --- | --- | --- | --- |
| `S5_OFF` | 電源鍵 | `S0_ON` | CPU、RAM、螢幕、風扇開啟 |
| `S0_ON` | 電源鍵 | `S5_OFF` | 全部關閉 |
| `S0_ON` | 低電量 | `S5_OFF` | 全部關閉 |
| `S0_ON` | 闔上螢幕 | `S3_SLEEP` | CPU、螢幕、風扇關閉，RAM 保持供電 |
| `S3_SLEEP` | 打開螢幕 | `S0_ON` | CPU、螢幕、風扇恢復供電 |
| `S3_SLEEP` | 電源鍵 | `S0_ON` | CPU、螢幕、風扇恢復供電 |
| `S3_SLEEP` | 低電量 | `S5_OFF` | 全部關閉 |

未定義的事件不會改變目前狀態。

## 編譯方式

請先確認系統已安裝 C 編譯器，例如 GCC、Clang 或 MinGW。

### Windows / MinGW

```sh

```

### Linux / macOS

```sh
gcc main.c -o main
```

### Windows

```sh
.\main.exe
```

### Linux / macOS

```sh

```

## 操作方式

程式啟動後，依照畫面提示輸入事件編號：

| 輸入 | 事件 |
| --- | --- |
| `1` | 按下電源鍵 |
| `2` | 闔上螢幕 |
| `3` | 打開螢幕 |
| `4` | 低電量 |
| `0` | 離開程式 |

範例流程：

```text
1  -> 從 S5 關機狀態進入 S0 開機狀態
2  -> 從 S0 開機狀態進入 S3 睡眠狀態
3  -> 從 S3 睡眠狀態喚醒回到 S0 開機狀態
4  -> 因低電量關機，回到 S5 關機狀態
```

## 設計概念

本專案將系統電源流程拆成兩個主要部分：

1. `SystemState`：描述目前系統所在的電源狀態。
2. `SystemEvent`：描述外部輸入事件。

狀態轉移集中在 `handle_state_machine()` 中處理，硬體供電則透過 `power_register` 的 bit 操作模擬。這種寫法可以清楚展示嵌入式系統常見的狀態機設計方式，也方便日後擴充更多狀態或事件。

## 可延伸方向

- 增加更多 ACPI 狀態，例如 `S4_HIBERNATE`。
- 模擬 AC 插拔、電池百分比、溫度過高等事件。
- 將狀態轉移表格化，降低 `switch-case` 複雜度。
- 加入單元測試驗證每個狀態轉移是否正確。
